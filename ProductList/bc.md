# bc — 命令行高精度计算器

## 📌 简介

| 项目 | 内容 |
|------|------|
| 开发者 | Philip A. Nelson (GNU) / Gavin D. Howard (现代实现) |
| 首次发布 | 1991 (POSIX bc) |
| 用途 | 任意精度的命令行计算器，支持数学函数和编程语法 |

## 🎯 用来做什么？

- **精确数学计算** — 浮点/整数任意精度，不会出现浮点舍入错误
- **进制转换** — 十进制、十六进制、二进制、八进制互转
- **脚本中的计算** — Shell 脚本里做数学运算（比 `$((...))` 强得多）
- **数学函数** — 正弦、余弦、对数、平方根等（需要 `-l` 加载数学库）

## 🔧 常用命令

| 命令 | 说明 |
|------|------|
| `echo "1+2" \| bc` | 基础四则运算 |
| `echo "scale=4; 10/3" \| bc` | 指定小数精度 |
| `echo "sqrt(144)" \| bc -l` | 平方根（-l 加载数学库） |
| `echo "obase=16; 255" \| bc` | 十进制转十六进制 |
| `echo "ibase=16; FF" \| bc` | 十六进制转十进制 |
| `echo "obase=2; 42" \| bc` | 十进制转二进制 |
| `bc -l` | 进入交互模式（可输入多条表达式） |

## 💡 日常使用示例

### 示例 1：精确浮点运算（Shell 默认做不到的）

```bash
# Shell 的 $((...)) 只支持整数
echo $((10 / 3))
# 输出: 3（被截断了！）

# bc 可以指定精度
echo "scale=4; 10 / 3" | bc
# 输出: 3.3333

# 更复杂的表达式
echo "scale=6; (5.2 * 3.7) / (2.1 + 0.9)" | bc
# 输出: 6.413333
```

### 示例 2：进制转换

```bash
# 十进制 → 十六进制
echo "obase=16; 255" | bc
# 输出: FF

echo "obase=16; 1024" | bc
# 输出: 400

# 十六进制 → 十进制
echo "ibase=16; FF" | bc
# 输出: 255

# 十进制 → 二进制
echo "obase=2; 42" | bc
# 输出: 101010

# 十进制 → 八进制
echo "obase=8; 255" | bc
# 输出: 377
```

> ⚠️ 注意：同时设置 `ibase` 和 `obase` 时，**先设 `obase`，再设 `ibase`**。否则 `ibase=16` 后，`obase=10` 中的 "10" 会被解读为十六进制（=16 十进制）。

### 示例 3：使用数学库函数

```bash
# -l 加载数学库，提供 sqrt, sin, cos, atan, log, exp 等

# 平方根
echo "sqrt(144)" | bc -l
# 输出: 12.00000000000000000000

# 正弦（弧度制）
echo "s(3.14159 / 2)" | bc -l
# 输出: .99999999999999999994（接近 1）

# 自然对数
echo "l(2.71828)" | bc -l
# 输出: .99999932734728219285（接近 1）
```

## 🚨 常见坑点

| 坑 | 原因 | 解决方法 |
|----|------|---------|
| `10 / 3` 输出 `3`（没有小数） | 默认 `scale=0`，只保留整数 | 设置 `scale=4` 等小数位数 |
| `ibase` 和 `obase` 同时设置时结果不对 | `ibase` 影响后面的数字解析 | 顺序写 `obase=16; ibase=10`（进制在前） |
| `bc -l` 后默认 `scale=20`，结果太长 | 数学库会设大精度 | 再设 `scale=4` 覆盖 |
| 指数运算用 `^`，不是 `**` | bc 语法与 C 不同 | 用 `^`：`echo "2^10" \| bc` |
| macOS 的 bc 和 Linux 略有差异 | 实现不同（Gavin Howard's bc vs GNU bc） | 基本功能一致，高级特性查 `man bc` |

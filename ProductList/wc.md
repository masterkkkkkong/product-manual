# wc — 文本统计工具

## 📌 简介

| 项目 | 内容 |
|------|------|
| 开发者 | Bell Labs（Unix Version 1） |
| 首次发布 | 1971 年 |
| 用途 | 统计文件的行数、单词数、字符数（字节数） |

## 🎯 用来做什么？

- **统计代码行数**——快速知道项目写了多少行代码
- **统计日志条目**——每行一条记录，`wc -l` 就是记录数
- **验证数据完整性**——导入导出前后对比行数是否一致
- **管道数据分析**——配合 `grep` 统计匹配了多少条

## 🔧 常用命令

| 命令 | 说明 |
|------|------|
| `wc file.txt` | 显示行数、单词数、字节数（三项） |
| `wc -l file.txt` | 只统计行数 |
| `wc -w file.txt` | 只统计单词数 |
| `wc -c file.txt` | 只统计字节数 |
| `wc -m file.txt` | 只统计字符数（区分多字节字符） |
| `wc *.py` | 统计当前目录所有 Python 文件，最后一行是总计 |
| `grep ERROR log.txt \| wc -l` | 统计日志中 ERROR 出现了多少次 |

## 💡 日常使用示例

### 示例 1：统计项目代码量

```bash
$ wc -l *.py

  120 scraper.py
   85 utils.py
  340 main.py
  545 total
```

### 示例 2：统计 CSV 数据行数（去掉表头）

```bash
$ cat sales.csv
date,product,amount
2024-01-01,apple,100
2024-01-02,banana,200
2024-01-03,cherry,150

$ tail -n +2 sales.csv | wc -l
3

# 3 条数据记录，不包含表头
```

### 示例 3：统计日志中的错误数量

```bash
$ cat app.log | grep "ERROR" | wc -l
23

# 过去一天的日志中有 23 条错误
```

## 🚨 常见坑点

| 问题 | 原因 | 解决 |
|------|------|------|
| `wc -c` 和 `wc -m` 结果不同 | `-c` 统计字节数，`-m` 统计字符数；中文字符占 3 字节 | 处理中文文本用 `wc -m`，处理二进制用 `wc -c` |
| 统计空文件显示 `0 0 0` | wc 对空文件的输出，没有特殊处理 | 用 `test -s file` 先判断文件是否为空 |
| `wc -l` 不统计最后一行如果没有换行符 | 某些编辑器保存文件不自动加末尾换行 | 用 `cat` 确认文件末尾是否有换行；脚本可用 `grep -c ''` 代替 |
| macOS 输出格式前面有空格 | BSD wc 默认输出带前导空格对齐 | 脚本处理用 `wc -l < file.txt`（无文件名输出）或 `awk '{print $1}'` 提取数字 |
